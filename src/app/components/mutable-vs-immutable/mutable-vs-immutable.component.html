W kontekście języka TypeScript i frameworka Angular, koncept mutable i immutable odnosi się do manipulacji danymi w aplikacji. Angular promuje korzystanie z niemutowalnych (immutable) danych w celu zwiększenia wydajności i uniknięcia nieprzewidywalnych efektów ubocznych. Oto kilka aspektów, które warto wziąć pod uwagę w Angularze:

  1. Niemutowalne struktury danych: W Angularze zaleca się korzystanie z niemutowalnych struktur danych, takich jak niemutowalne tablice (Immutable Arrays) lub niemutowalne obiekty (Immutable Objects). Można to osiągnąć poprzez użycie metod i operatorów takich jak map(), filter(), concat(), spread operator (...) itp., które tworzą nowe niemutowalne struktury danych zamiast modyfikować istniejące.

  2. OnPush Change Detection: W przypadku komponentów Angular, które korzystają z strategii detekcji zmian "OnPush" (OnPush Change Detection), zmiany w danych powinny być wykrywane tylko wtedy, gdy referencja do danych ulegnie zmianie. Zatem zamiast bezpośrednio modyfikować istniejące dane, zaleca się tworzenie nowych niemutowalnych obiektów i przypisywanie ich do właściwości komponentu w celu wywołania procesu detekcji zmian.

  3. Niezmienna reakcja: Przy pracy z Reactive Extensions (RxJS) w Angularze, zaleca się korzystanie z operatorów RxJS, które promują niemutowalność danych, takich jak map(), filter(), scan() itp. Te operatory tworzą nowe struktury danych, zachowując niemutowalność.

Przykład:

typescript
import {{'{'}} Component, ChangeDetectionStrategy } from '@angular/core';
import {{'{'}} ImmutableArray } from 'immutable'; // Przykładowa biblioteka obsługująca niemutowalne tablice

@Component({{'{'}}
  selector: 'app-example',
  template: `
    <ul>
      <li *ngFor="let item of items">{{'{'}}{{'{'}} item }}</li>
    </ul>
  `,
  changeDetection: ChangeDetectionStrategy.OnPush // Strategia detekcji zmian OnPush
})
export class ExampleComponent {{'{'}}
  items: ImmutableArray<string> = ImmutableArray(['A', 'B', 'C']); // Niemutowalna tablica

  addItem(): void {{'{'}}
    this.items = this.items.push('D'); // Tworzenie nowej niemutowalnej tablicy z nowym elementem
  }
}

W powyższym przykładzie używamy niemutowalnej tablicy z biblioteki "immutable" i korzystamy z metody push(), która tworzy nową niemutowalną tablicę z dodanym elementem.

Korzystanie z niemutowalnych danych w Angularze pomaga uniknąć problemów związanych z mutacją danych, ułatwia debugowanie i wpływa pozytywnie na wydajność aplikacji poprzez redukcję zbęd
