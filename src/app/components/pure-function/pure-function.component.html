Pure function (czysta funkcja) to funkcja, która zawsze zwraca tę samą wartość dla tych samych argumentów i nie powoduje żadnych efektów ubocznych. Oznacza to, że jej działanie jest w pełni deterministyczne i nie zależy od żadnych stanów zewnętrznych.

Oto kilka cech charakterystycznych dla czystych funkcji:

  1. Predictability (Przewidywalność): Czysta funkcja zawsze zwraca ten sam wynik dla tych samych argumentów. Brak zależności od jakichkolwiek zmiennych globalnych, stanu aplikacji lub zewnętrznych zasobów.

  2. No Side Effects (Brak efektów ubocznych): Czysta funkcja nie wpływa na stan zewnętrzny aplikacji ani nie powoduje żadnych zmian w zmiennych poza swoim zakresem. Nie wykonuje operacji takich jak zapis do plików, zapytania do bazy danych czy manipulacje DOM.

  3. Immutability (Niemutowalność): Czysta funkcja nie modyfikuje swoich argumentów. Jeśli wykonuje operacje na danych, tworzy nowe obiekty lub struktury danych zamiast zmieniać te istniejące.

  4. Composability (Komponowalność): Czyste funkcje są łatwe do komponowania i łączenia w większe struktury, ponieważ nie mają zależności od stanu zewnętrznego. Można je łączyć w łańcuchy wywołań, tworzyć funkcje wyższego rzędu i tworzyć bardziej modułowe i elastyczne systemy.

Przykład czystej funkcji w języku JavaScript:

javascript
function add(a, b) {{'{'}}
  return a + b;
}

Funkcja add jest czysta, ponieważ zawsze zwraca sumę dwóch liczb a i b i nie wpływa na żadne zmienne zewnętrzne ani stan aplikacji.

Korzystanie z czystych funkcji ma wiele korzyści, takich jak łatwiejsze testowanie, czytelniejszy kod, unikanie błędów związanych z efektami ubocznymi oraz ułatwienie zrozumienia i utrzymania kodu. W programowaniu funkcyjnym czyste funkcje są kluczowym konceptem, promującym odizolowanie i modułowość kodu.
