Mixin-y w preprocesorach CSS to narzędzia, które umożliwiają tworzenie wielokrotnie używalnych bloków reguł stylów. Dzięki nim można zdefiniować zestaw właściwości i wartości stylów, które można następnie wykorzystać w wielu miejscach kodu.

Oto przykład użycia mixin-a w Sass/SCSS:

scss
@mixin button-style {{'{'}}
        display: inline-block;
        padding: 10px 20px;
        background-color: #007bff;
        color: #fff;
        border: none;
        border-radius: 4px;
        font-size: 16px;
        text-decoration: none;
      }
      
      .button {{'{'}}
        @include button-style;
      }
      
      .submit-button {{'{'}}
        @include button-style;
        background-color: #28a745;
      }

W powyższym przykładzie zdefiniowano mixin o nazwie button-style, który zawiera zestaw reguł stylów dla przycisków. Następnie mixin jest wykorzystywany w dwóch różnych selektorach .button i .submit-button. Dodatkowo w przypadku .submit-button nadpisano kolor tła.

Po skompilowaniu tego kodu do zwykłego CSS, otrzymamy:

css
.button {{'{'}}
        display: inline-block;
        padding: 10px 20px;
        background-color: #007bff;
        color: #fff;
        border: none;
        border-radius: 4px;
        font-size: 16px;
        text-decoration: none;
      }
      
      .submit-button {{'{'}}
        display: inline-block;
        padding: 10px 20px;
        background-color: #28a745;
        color: #fff;
        border: none;
        border-radius: 4px;
        font-size: 16px;
        text-decoration: none;
      }

Dzięki mixin-om możemy uniknąć powtarzania kodu CSS i zachować spójność w naszych stylach. Możemy również przekazywać argumenty do mixin-ów, co pozwala na jeszcze większą elastyczność i reużywalność.
