W kontekście Angulara, termin "pure pipe" odnosi się do specjalnego rodzaju pipe, który nazywany jest również "czystym" (pure). Pure pipe to taki , który wykonuje czyste, niemutowalne operacje na danych wejściowych i zwraca nowe dane bez zmiany oryginalnych danych wejściowych. Innymi słowy, pure pipe nie modyfikuje danych wejściowych, a jedynie przetwarza je i zwraca przekształcone dane wyjściowe.

Aby zaimplementować pure pipe w Angularze, należy zastosować dekorator @Pipe wraz z parametrem pure ustawionym na true. Przykład deklaracji pure pipe wygląda następująco: 

typescript
import {{'{'}} Pipe, PipeTransform } from '@angular/core';

        @Pipe({{'{'}}
          name: 'myPurePipe',
          pure: true
        })
        export class MyPurePipe implements PipeTransform {{'{'}}
          transform(value: any, args?: any): any {{'{'}}
            // Przetwarzanie danych wejściowych i zwracanie przekształconych danych
            // bez modyfikacji danych wejściowych
          }
        }

Warto zauważyć, że pure pipes w Angularze są wywoływane tylko wtedy, gdy wartość danych wejściowych zmieni się na podstawie referencji. Dlatego, jeśli dane wejściowe są obiektami lub tablicami, a ich zawartość zostanie zmieniona bez zmiany samej referencji, pipe nie zostanie automatycznie uruchomiony. W takich przypadkach konieczne może być ręczne wywołanie pipe przy użyciu metody transform w celu przetworzenia danych i zaktualizowania widoku.

Ogólnie rzecz biorąc, stosowanie pure pipes w Angularze ma na celu optymalizację wydajności aplikacji poprzez unikanie niepotrzebnego przetwarzania danych, gdy nie występują zmiany w danych wejściowych.
