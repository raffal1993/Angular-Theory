const i let to dwa słowa kluczowe w języku JavaScript, które służą do deklarowania zmiennych. Oto różnice między nimi:

  1. Przypisanie wartości:
  • const: Zmienne zadeklarowane za pomocą const są przypisywane wartością, która nie może być zmieniona po zainicjalizowaniu. Oznacza to, że po przypisaniu wartości do zmiennej const, nie można jej już zmienić.
  • let: Zmienne zadeklarowane za pomocą let są przypisywane wartością, która może być później zmieniona. Można przypisywać nowe wartości do zmiennych let w dowolnym miejscu w bloku, w którym zostały zadeklarowane.

Przykład:

javascript
const age = 30;
        age = 40; // Błąd, próba zmiany wartości zmiennej const
        
        let name = 'John';
        name = 'Jane'; // Poprawne, można zmieniać wartość zmiennej let
  1. Zasięg:
  • const i let mają blokowy zasięg, co oznacza, że są widoczne tylko w bloku, w którym zostały zadeklarowane. Blok to sekcja kodu ograniczona przez nawiasy klamrowe {{ "{" }}}.
  • Zmienne const i let nie są hoistowane, co oznacza, że muszą być zadeklarowane przed użyciem wewnątrz bloku.

Przykład:

javascript
{{'{'}}
          const x = 10;
          let y = 20;
        
          console.log(x); // Poprawne, zmienna const widoczna w bloku
          console.log(y); // Poprawne, zmienna let widoczna w bloku
        }
        
        console.log(x); // Błąd, zmienna const niedostępna poza blokiem
        console.log(y); // Błąd, zmienna let niedostępna poza blokiem
  1. Ponowna deklaracja:
  • const nie pozwala na ponowną deklarację zmiennej o tej samej nazwie w tym samym zasięgu. Każda zmienna const musi mieć unikalną nazwę w danym zasięgu.
  • let pozwala na ponowną deklarację zmiennej w tym samym zasięgu, ale tylko jeśli jest to w innych blokach. Nie można ponownie deklarować zmiennej let w tym samym bloku.

        let y = 20;

          let y = 40; // Poprawne, inny blok, inna zmienna let
        }
        
        console.log(y); // Poprawne, wartość y poza blokiem if
