Web Components to zestaw standardów webowych, które umożliwiają tworzenie i ponowne wykorzystywanie własnych komponentów interfejsu użytkownika w sposób niezależny od frameworków czy bibliotek JavaScript. Składają się one z trzech głównych technologii:

  1. Custom Elements: Custom Elements umożliwiają tworzenie własnych elementów HTML o niestandardowych nazwach i zachowaniu. Dzięki temu można tworzyć własne komponenty, takie jak <my-button> czy <my-carousel>, które mogą być używane jak dowolne standardowe elementy HTML.

  2. Shadow DOM: Shadow DOM pozwala na izolację stylów i struktury DOM wewnątrz danego komponentu. Każdy komponent może posiadać swój własny Shadow DOM, w którym definiuje się style CSS i strukturę HTML. To zapobiega konfliktom stylów i skrywa wewnętrzną implementację komponentu.

  3. HTML Templates: HTML Templates pozwalają na tworzenie szablonów HTML, które można zduplikować i używać wielokrotnie w różnych miejscach. Szablony te nie są renderowane na stronie do momentu, aż zostaną użyte przez skrypt JavaScript, co umożliwia dynamiczne tworzenie i wstawianie komponentów.

Web Components są zgodne ze standardami W3C i są obsługiwane przez większość nowoczesnych przeglądarek. Dzięki nim możemy tworzyć hermetyczne, niezależne od siebie komponenty, które można używać w różnych projektach i frameworkach.

Niezależność od frameworków daje elastyczność i umożliwia ponowne wykorzystywanie komponentów w różnych kontekstach. Możemy także integrować Web Components z istniejącym kodem JavaScript czy frameworkami, co czyni je uniwersalnym rozwiązaniem do tworzenia modułowych i skalowalnych interfejsów użytkownika.
