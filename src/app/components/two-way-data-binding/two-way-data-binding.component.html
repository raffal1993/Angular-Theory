Two-way data binding in Angular allows for synchronization of data between the component and the template in both directions. It means that changes in the component can automatically update the template, and vice versa, changes in the template can update the component.

In Angular, two-way data binding is typically achieved using the [(ngModel)] syntax, which combines property binding and event binding. Here's how it works:

  1. Import the FormsModule: First, ensure that the FormsModule is imported in your Angular module file (app.module.ts or relevant module file). This module is required for using two-way data binding with ngModel. 
typescript
import {{'{'}} FormsModule } from '@angular/forms';

        @NgModule({{'{'}}
          imports: [
            FormsModule
          ],
          // Other module configurations...
        })
        export class AppModule {{'{'}} }
  1. Component property and template binding: In your component class, declare a property and bind it to an input field in the template using [(ngModel)]: 
typescript
@Component({{'{'}}
          selector: 'app-example',
          template: `
            <input [(ngModel)]="name" placeholder="Enter your name">
            <p>Your name: {{'{'}}{{'{'}} name }}</p>
          `
        })
        export class ExampleComponent {{'{'}}
          name: string = '';
        }

In the above example, the name property of the component is bound to the input field using [(ngModel)]. Any changes made in the input field will automatically update the name property, and vice versa.

  1. Enable two-way binding with ngModel directive: To enable two-way binding with ngModel, you need to import the FormsModule and use the ngModel directive on the input element. Make sure to include the name attribute on the input element as well. 
html
<input [(ngModel)]="name" name="name" placeholder="Enter your name">

By using [(ngModel)], Angular will automatically handle the synchronization between the component and the template, updating the name property whenever the input field changes, and updating the input field whenever the name property changes.

It's important to note that two-way data binding requires the FormsModule to be imported, and it works with form controls, such as input fields, checkboxes, radio buttons, etc.

Additionally, if you're using Angular version 10 or later, you can use the [(ngModel)] syntax without explicitly importing the FormsModule, as it's included by default in the CommonModule.
