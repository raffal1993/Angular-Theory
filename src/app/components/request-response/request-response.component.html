Żądanie (Request): Żądanie (request) jest wysyłane przez klienta (client) do serwera (server) w celu wykonania określonej akcji lub pobrania zasobu. Składa się z następujących elementów:

  1. Metoda HTTP (HTTP method): Określa rodzaj operacji do wykonania, na przykład GET, POST, PUT, PATCH, DELETE itp.
  2. Adres URL (Uniform Resource Locator): Określa adres zasobu lub punktu końcowego na serwerze.
  3. Nagłówki (Headers): Zawierają dodatkowe informacje o żądaniu, takie jak typ zawartości (content type), uwierzytelnianie (authentication) i dyrektywy dotyczące buforowania (caching directives).
  4. Ciało (Body) (opcjonalne): Zawiera dane wysyłane do serwera, zwykle używane w żądaniach typu POST lub PUT do przekazania danych.

Odpowiedź (Response): Odpowiedź (response) to odpowiedź serwera na żądanie klienta. Zawiera informacje o statusie żądania oraz wszelkie żądane dane lub zasoby. Odpowiedź zwykle składa się z następujących elementów:

  1. Linia statusu (Status Line): Określa kod stanu (status code) i krótki opis tekstowy wyniku odpowiedzi.
  2. Nagłówki (Headers): Zapewniają dodatkowe informacje o odpowiedzi, takie jak typ zawartości (content type), dyrektywy buforowania (caching directives) i szczegóły serwera (server details).
  3. Ciało (Body) (opcjonalne): Zawiera rzeczywiste dane lub żądane zasoby, takie jak treść HTML, dane JSON lub zawartość pliku.

Cykl żądanie-odpowiedź (Request-Response cycle) stanowi podstawę komunikacji między klientami a serwerami w różnych technologiach webowych, interfejsach API i protokołach sieciowych, umożliwiając wymianę danych i interakcję między różnymi komponentami systemu rozproszonego.
